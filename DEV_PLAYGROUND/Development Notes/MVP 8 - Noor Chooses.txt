## ‚úÖ Step 2: Add a Preference Function

```python
import random

def prefer_motif(options: list[str]) -> str:
    # Simple weighted preference, can be made more complex
    weights = {
        "bittersweet": 0.7,
        "tenderness": 0.3,
        "melancholy": 0.6,
        "isolation": 0.4,
        "awe": 0.5,
        "peace": 0.5
    }
    scored = [(m, weights.get(m, 0.5) + random.uniform(-0.1, 0.1)) for m in options]
    scored.sort(key=lambda x: x[1], reverse=True)
    return scored[0][0]
```

This gives Noor **gentle variability** ‚Äî she leans toward motifs, but doesn‚Äôt always repeat.

## TROUBLESHOOTING

## ü©π What Needs Fixing

This line:

```python
if rule["requires"].issubset(seen_motifs) and rule not in fired_rules:
```

Will **fail** to prevent repeat firing because `rule` is a **dictionary**, and dictionaries are not hashable (you can‚Äôt store them in a set directly).

As a result, the same rule may fire **multiple times** ‚Äî even though it looks like it‚Äôs tracked.

---

## ‚úÖ The Fix

Use a **frozen representation** of the rule‚Äôs required motif set (like we discussed before):

### üîÅ Replace this:

```python
fired_rules = set()
...
for rule in INFERENCE_RULES:
    if rule["requires"].issubset(seen_motifs) and rule not in fired_rules:
```

### ‚úÖ With this:

```python
fired_rules = set()
...
for rule in INFERENCE_RULES:
    rule_key = frozenset(rule["requires"])
    if rule_key not in fired_rules and rule["requires"].issubset(seen_motifs):
```

### üîÑ Then replace:

```python
fired_rules.add(rule)
```

‚Üí with:

```python
fired_rules.add(rule_key)
```

---

## üîç Why This Works

By converting `rule["requires"]` (a `set`) into a `frozenset`, you make it:

* ‚úÖ **Hashable**
* ‚úÖ **Storable in a set**
* ‚úÖ **Uniquely tied to that motif combination**

This ensures **only one inference** per rule condition ‚Äî *true choice*.
